### WORKSPACE_DIR 陷阱（关键教训）
- `agent/internal/common/config.go` 用 `godotenv.Load()` 加载 `.env`
- `.env` 里的 `WORKSPACE_DIR=` 覆盖 shell 环境变量
- `inferWorkspaceDir()` fallback = `/root/vibex-workbench`
- **结论**：不要靠 env 传 workspace_root，API handler 已从请求体读取
- 测试空目录：`rm -rf /tmp/vibex-e2e-test && mkdir /tmp/vibex-e2e-test`
